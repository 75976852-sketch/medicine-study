## 方案一：最简单 - 使用 Vercel（推荐新手）

### ✅ 优点
- **完全免费**
- **5分钟搞定**
- **自动配置 HTTPS（安全锁）**
- **不需要服务器**
- **自动部署更新**

### 📝 操作步骤

#### 第1步：准备文件
确保你的文件夹里有这三个文件：
- `index.html`
- `style.css`
- `script.js`

✅ 检查方法：在 Finder（Mac）或文件管理器（Windows）里打开你的项目文件夹，能看到这三个文件就行。

#### 第2步：注册 Vercel 账号
1. 打开浏览器，访问：https://vercel.com
2. 点击右上角 **"Sign Up"**（注册）
3. 建议选择 **"Continue with GitHub"**（用 GitHub 账号登录）
   - 如果没有 GitHub，先注册一个：https://github.com
   - 或者用邮箱注册也可以

#### 第3步：上传代码到 GitHub（如果还没有）
**如果已经有 GitHub 仓库，跳过这一步**

1. 打开 https://github.com，登录
2. 点击右上角 **"+"** → **"New repository"**（新建仓库）
3. 填写信息：
   - Repository name（仓库名）：`medicine-study`（随便起名）
   - 选择 **Public**（公开）
   - 不要勾选任何选项
   - 点击 **"Create repository"**
4. 上传代码（两种方法选一种）：

**方法A：网页上传（最简单）**
1. 在新建的仓库页面，找到 **"uploading an existing file"**（上传已有文件）
2. 点击进入
3. 把你项目文件夹里的所有文件拖进去（除了 `.git` 文件夹）
4. 点击 **"Commit changes"**

**方法B：使用 Git（稍微复杂）**
如果你电脑上安装了 Git，可以这样做：
```bash
cd /Users/liyuange/tranditional\ chineses
git init
git add .
git commit -m "初始提交"
git branch -M main
git remote add origin https://github.com/你的用户名/medicine-study.git
git push -u origin main
```

#### 第4步：在 Vercel 部署
1. 登录 Vercel 后，点击 **"Add New..."** → **"Project"**（项目）
2. 在 **"Import Git Repository"** 下面，找到你的 GitHub 仓库（medicine-study）
3. 点击 **"Import"**（导入）
4. 直接点击 **"Deploy"**（部署），不用改任何设置
5. 等待 1-2 分钟，看到 **"Congratulations"** 就成功了！

#### 第5步：获取网站地址
部署成功后，你会看到一个地址，比如：
```
https://medicine-study-xxxxx.vercel.app
```
这就是你的网站地址！点击就能访问。

#### 第6步：绑定自己的域名（可选）

**如果你有域名，想用 `yourname.com` 访问：**

1. 在 Vercel 项目页面，点击 **"Settings"**（设置）
2. 点击左侧 **"Domains"**（域名）
3. 在输入框输入你的域名，比如 `medicine.yourname.com`
4. 点击 **"Add"**（添加）
5. Vercel 会告诉你需要配置 DNS，按照提示：
   - 登录你的域名管理后台（比如阿里云、腾讯云、Namecheap）
   - 添加一条 **CNAME 记录**：
     - 主机记录：`medicine`（或 `@` 表示主域名）
     - 记录值：Vercel 给你的地址（比如 `cname.vercel-dns.com`）
6. 等待 5-30 分钟，DNS 生效后就能用你的域名访问了！

## 方案二：使用自己的服务器 - Nginx

### ⚠️ 前提条件
- 有一台服务器（阿里云、腾讯云、VPS等）
- 服务器安装了 Ubuntu 或 CentOS 系统
- 知道服务器的 IP 地址和密码

### 📝 详细步骤

#### 第1步：连接服务器

**Mac/Linux 用户：**
打开终端（Terminal），输入：
```bash
ssh root@你的服务器IP
# 例如：ssh root@123.456.789.0
```
输入密码（输入时不会显示，直接回车）

**Windows 用户：**
1. 下载 PuTTY：https://www.putty.org/
2. 打开 PuTTY
3. 输入服务器 IP，点击 Open
4. 输入用户名（通常是 `root`）和密码

#### 第2步：安装 Nginx

连接到服务器后，在终端输入：

**Ubuntu/Debian 系统：**
```bash
# 更新系统
sudo apt update

# 安装 Nginx
sudo apt install nginx -y

# 启动 Nginx
sudo systemctl start nginx

# 设置开机自启
sudo systemctl enable nginx
```

**CentOS 系统：**
```bash
# 安装 Nginx
sudo yum install nginx -y

# 启动 Nginx
sudo systemctl start nginx

# 设置开机自启
sudo systemctl enable nginx
```

**测试是否安装成功：**
在浏览器访问 `http://你的服务器IP`，如果看到 Nginx 欢迎页面，说明安装成功！

####  第3步：上传文件到服务器

**方法A：使用 SCP（Mac/Linux）**

在你的**本地电脑**的终端（不是服务器），输入：
```bash
cd /Users/liyuange/tranditional\ chineses

# 上传所有文件到服务器
scp -r * root@你的服务器IP:/var/www/medicine-study/

# 例如：scp -r * root@123.456.789.0:/var/www/medicine-study/
```
输入服务器密码

**方法B：使用 FTP 工具（推荐新手）**

1. 下载 FileZilla：https://filezilla-project.org/
2. 安装并打开 FileZilla
3. 输入信息连接服务器：
   - 主机：`sftp://你的服务器IP`
   - 用户名：`root`
   - 密码：你的服务器密码
   - 端口：`22`
4. 连接成功后：
   - 左边窗口：找到你的项目文件夹（tranditional chineses）
   - 右边窗口：进入 `/var/www/` 目录，新建文件夹 `medicine-study`
   - 把左边所有文件拖到右边的 `medicine-study` 文件夹

#### 第4步：配置 Nginx

1. 在服务器终端输入，创建配置文件：
```bash
sudo nano /etc/nginx/sites-available/medicine-study
```

2. 复制下面的内容，粘贴进去（按 `Ctrl+Shift+V` 粘贴）：
```nginx
server {
    listen 80;
    server_name 你的域名.com www.你的域名.com;  # 改成你的域名，没有域名就改成你的服务器IP
    
    root /var/www/medicine-study;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

3. 保存并退出：
   - 按 `Ctrl+X`
   - 按 `Y`（确认）
   - 按 `Enter`（回车）

4. 创建软链接：
```bash
sudo ln -s /etc/nginx/sites-available/medicine-study /etc/nginx/sites-enabled/
```

5. 删除默认配置（避免冲突）：
```bash
sudo rm /etc/nginx/sites-enabled/default
```

6. 测试配置是否正确：
```bash
sudo nginx -t
```
应该显示 `syntax is ok` 和 `test is successful`

7. 重启 Nginx：
```bash
sudo systemctl restart nginx
```

#### 第5步：设置文件权限
```bash
sudo chown -R www-data:www-data /var/www/medicine-study
sudo chmod -R 755 /var/www/medicine-study
```

#### 第6步：配置防火墙（开放80端口）
```bash
# Ubuntu/Debian
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw reload

# CentOS
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
```

#### 第7步：测试访问

在浏览器访问：
- 如果有域名：`http://你的域名.com`
- 如果没有域名：`http://你的服务器IP`

如果能看到你的网站，说明部署成功！

#### 第8步：配置 HTTPS（安全锁，推荐）

1. 安装 Certbot：
```bash
# Ubuntu/Debian
sudo apt install certbot python3-certbot-nginx -y

# CentOS
sudo yum install certbot python3-certbot-nginx -y
```

2. 获取证书（**需要有域名才能用这个**）：
```bash
sudo certbot --nginx -d 你的域名.com -d www.你的域名.com
```

3. 按照提示操作：
   - 输入邮箱地址
   - 同意服务条款（输入 `Y`）
   - 是否分享邮箱（输入 `N` 或 `Y` 都可以）
   - 选择重定向 HTTP 到 HTTPS（输入 `2`）

4. 完成后，访问 `https://你的域名.com` 就能看到安全锁了！

5. 证书会自动续期，不用担心过期。

## 常见问题解答

### ❓ 问题1：我是小白，应该选哪个方案？

**答：** 强烈推荐**方案一（Vercel）**！
- 最简单，5分钟搞定
- 完全免费
- 不需要懂服务器
- 自动配置 HTTPS

### ❓ 问题2：我没有域名怎么办？

**答：**
- **Vercel**：会给你一个免费地址，比如 `medicine-study.vercel.app`
- **服务器**：可以用服务器 IP 直接访问，但建议买个域名（很便宜，一年几十块）

### ❓ 问题3：Vercel 部署后，我想更新网站怎么办？

**答：**
1. 在 GitHub 更新代码
2. Vercel 会自动检测并重新部署
3. 等1-2分钟，网站就更新了！

### ❓ 问题4：用服务器部署，更新网站怎么办？

**答：**
重新上传文件到服务器，覆盖旧文件：
```bash
scp -r * root@你的服务器IP:/var/www/medicine-study/
```
然后访问网站就能看到更新。

### ❓ 问题5：网站打不开怎么办？

**检查清单：**
1. ✅ 服务器是否运行正常？
2. ✅ Nginx 是否启动？运行 `sudo systemctl status nginx` 查看
3. ✅ 文件是否上传到正确位置？检查 `/var/www/medicine-study/` 里有没有文件
4. ✅ 防火墙是否开放80端口？
5. ✅ 域名 DNS 是否解析正确？（用 `ping 你的域名` 测试）

### ❓ 问题6：数据会丢失吗？

**答：** 
- 这个应用使用浏览器的 localStorage 存储数据
- 每个用户的数据都存在他们自己的浏览器里
- 服务器上不存储用户数据
- **但是**：用户换浏览器或清除缓存，数据会丢失
- 如果需要跨设备同步，需要开发后端功能（后续可添加）

### ❓ 问题7：可以多人同时使用吗？

**答：** 可以！因为是前端应用，理论上可以无限多用户同时使用。
